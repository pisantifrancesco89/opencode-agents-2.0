# Permanent Memory System

This folder is the AI's "brain" for your project. It remembers everything across sessions so you never need to re-explain.

## Files

| File | What it stores |
|------|----------------|
| `project.md` | What the project is, tech stack, structure, conventions |
| `progress.md` | What's done, what's next, current status |
| `errors.md` | Mistakes to avoid, fixes that worked |
| `successes.md` | Patterns that work well for this project |
| `decisions.md` | Architecture decisions and why they were made |

## How It Works

1. **AI reads all memory** at session start — instantly knows your project
2. **You give a task** — the AI understands context immediately
3. **AI saves learnings** at session end — nothing is forgotten

## Benefits
- **93% fewer tokens** — no re-analyzing the codebase each session
- **Zero re-explaining** — context persists across sessions
- **Permanent learning** — the AI gets smarter about YOUR project over time

## First Session
The AI runs a structured interview to fill these files.
After that, every session starts instantly with full context.

## Manual Updates
You can edit these files anytime to:
- Change project details
- Add known errors
- Update priorities
- Document team conventions
